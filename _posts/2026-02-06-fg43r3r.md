시네머신


시네머신이란?
  - 카메라 선택과 전환을 자동화한 기능
  - 카메라에 어떤 규칙을 부여해서 활용하는 시스템

시네머신의 이점
  - 카메라 로직이 분리
  - 수학적 지식이 크게 요구되지 않음
  - 카메라 상태 관리 가능

  - 상황별 연출을 코드 없이 구현 가능
  - 플레이어 이동이나 상태 변화를 카메라에 자연스럽게 적용 가능


카메라 추적 코드
  - ```csharp
      void LateUpdate()
      {
        Camera.main.transform.position = player.position + offset;

        Camera.main.transform.LookAt(player);
      }
    ```
  - 왜 이런 카메라 추적 코드 방식을 잘 사용하지 않을까?
    - 하드 코딩 로직
    - 예외 처리 지옥
    - 상태 전환 불가능 (기획이 바뀌면 하드 코딩 로직과 예외 처리 지옥 때문에 상태 전환 하기 힘들고 코드 다시 짜야함)


시네머신 구성
  - 브레인
  - 버츄얼 캠(가상 카메라)

시네 머신 브레인(감독)
  - 어떤 가상 카메라를 이용할지 결정
  - 블랜딩(전환 효과) 방식을 결정

  -  실제 카메라는 브레인만 보고, 브레인은 가상 카메라 중 하나를 고름

버츄얼 카메라
  - VC, 브이캠
  - 하는 일
    - 누구를 따라갈 것인지 (Follow)
    - 어디를 볼 것인지 (Look At)
    - 어느 거리 / 각도 / 부드러움 규칙을 설정

  계층 구조
    - ```
        Main Camera(메인 카메라)
          Cinemachine Brain
            Virtual Camera A
            Virtual Camera B
            Virtual Camera C
            ...
      ```
    - 같은 우선순위면 가장 최근에 활성화된 카메라를 우선 순위 높게

  - 브이캠의 속성
    - 대상 설정 : Follow & LookAt
      - Follow : 어떤 대상의 몸체를 매칭
      - LookAt : 기획에 따라 다르지만, 따로 빈 게임 오브젝트로 빼서 설정함


블랜드 리스트 카메라
  - 여러 브이캠을 리스트로 묶음

- 사례
  - 인트로 연출
  - 컷신 시작
  - 보스 등장


클리어 샷 카메라

돌리캠

- 구성 및 특징
