리스트와 링크드리스트


리스트

List<T>의 동작 원리와 사용법을 이해하고 설명할 수 있다.
LinkedList<T>의 동작 원리와 사용법을 이해하고 설명할 수 있다.
순차 자료구조와 연결 자료구조의 차이점에 대해 이해하고 설명할 수 있다.


List<string> list = new();


// 삽입
// 맨 뒤에 데이터 추가 시
// 중간이나 맨 앞에 

list.Add("e");
list.Add(2,"c");

list.insert();

list.contains();

list.Remove();
list.RemoveAt(0);

list.sort();    // 낮은 숫자부터 정렬


capacity 내부의 전체 크기
list.count 실제 데이터가 들어간 공간의 크기


리스트는 검색을 순회하면서 특정 데이터가 있는지 확인, O(n) 복잡도

리스트는 읽을 때 내부는 배열로 구서오디어 연속된 메모리 공간에 위치하므로 인덱스로 접근 가능, O(1)의 복잡도

리스트도 클래스





링크드 리스트

순차 자료구조와는 다르게 메모리의 공간 상에 연속적으로 위치하지 않고 랜덤한 위치에 존재하는 자료구조

단일 연결 리스트
 원형 연결 리스트
 이중 연결 리스트
 우너형 이중 연결 리스트
    LinkedList<T>

서로 떨어져있어서 인덱스로 접근할 수 없음
읽기 검색 O(n) 의 복잡도
읽기와 검색 list 보다 느림
대신 참조만 연결해주면 되어서, 삽입 삭제 시 상대적으로 용이함
