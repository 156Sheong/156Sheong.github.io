## B. 



게임 프로젝트 개발 시 생각해야할 것
    1. 무슨 게임을 만드는가?
    2. 유저가 보게 될 결과물(화면)은 무엇인가?
    3. 최소 기능들이 무엇이 있는데?
    4. 우선 순위가 무엇인가?


게임 개발 순서
  1. 레퍼런스 게임 플레이
  2. 해당 게임의 기능을 쪼개고 나열하기
     - 각 기능은 동사로 끝나는 한 문장으로 적어야 함
  3. 나열한 기능을 역할(책임)에 맞춰 묶기(클래스 만들기)
  4. 연관있는 클래스 끼리 연결하기(누구와 참조되는가)
  5. 실제 코딩하기


소코반 게임의 기능들

게임을 시작한다
메뉴를 선택한다
게임을 종료한다

스테이지에 맞게 맵을 만든다
맵을 초기화하여 재시작한다

인풋을 받는다

캐릭터가 이동한다
캐릭터가 아이템과 충돌한다
캐릭터가 벽이나 장애물에 막힌다
캐릭터가 사망한다


캐릭터가 아이템과 상호작용한다
  캐릭터가 아이템을 밀어낸다
아이템과 아이템이 상호작용한다

화면에 결과를 출력한다

승리 / 패배 결과를 판단한다
다음 맵으로 이동한다



소코반 게임의 기능 묶음
오브젝트
맵
  오브젝트 초기화 및 업데이트
플레이어
  입력 및 이동
출력
  가이드
  이동 횟수
  골 갯수
사운드
데이터



GameManager.cs
  - 게임이 실시간으로 돌아가는 것
InputManager.cs
  - 플레이어 외 입력 받기
Map.cs
  - 맵 초기화 및 업데이트
Player.cs
  - 플레이어 입력, 이동 및 상호작용
Object.cs
  - 오브젝트 속성
  - (콘솔 소코반 규모에선 크게 사용할 일은 없음)
Interect.cs
  - 오브젝트 간 상호작용
Rules.cs
  - 승 / 패 규칙
Render.cs
  - 게임의 상태 출력
  - (부가 기능)
SoundManager.cs
  - 각종 음악 재생
