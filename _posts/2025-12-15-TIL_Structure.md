  구조체는 연속된 메모리 공간 사용
  구조체 내부 가장 큰 변수 타입에 맞춰 메모리 할당



배열의 값은 힙 데이터에 저장되지만, 주소는 스택 메모리에 연속되어 저장되듯,
구조체는 값타입이니까 스택 메모리에 값이 저장될 텐데, 이 때 같은 구조체 내부의 데이터는 메모리에 연속되어 저장되나요?

-> 값타입이라고 꼭 스텍에 저장되는 건 아님, 상황에 따라 다른데 이는 클래스를 배워야 함, 연속되어 저장되는건 맞음


아까의 질문과 연속해서 질문이 있습니다.
혹시 메모리에서 배열의 첫 주소(참조 타입)만 알면 나머지 배열 주소를 알 수 있듯,
구조체도 첫번째 값(값타입)이 메모리 어디 적혀있는지만 알면, 그 뒤에 나올 값들은 메모리 어디에 들어있는지 파악이 가능할까요??
공간이 연속되었다면 파악하기 용이할 것 같은데,
구조체는 배열과 다르게 각자 차지하는 메모리 공간이 다를 수 있을 것 같아서(bool, byte, double 이 같은 구조체에 있을 수 있으니)
혹시 값의 위치가 파악이 안될까 싶기도 해서 여쭤봅니다.

중첩 구조체

멤버 변수



class Warrior // 이건 클레스이지만, 구조체도 비슷하게 작동
{
    public string Name; // 이름
    public int Hp;      // 체력
}

생성자가 없을 시
Warrior myCharacter = new Warrior(); // 구조체의 변수 값 할당 없이 전사 캐릭터 생성 ()

myCharacter.Name = "전사"; 
myCharacter.Hp = 100;
이렇게 따로 값 입력해야, 까먹으면 안 됨

생성자가 있을 시
Warrior myCharacter = new Warrior("전사", 100); // 구조체의 변수 값 할당을 강제로 해야하고, 안 넣으면 에러를 발생시킴

까먹으면 쉽게 확인 가능


비유가 이렇네요
