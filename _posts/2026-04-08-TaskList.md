## B. 작업 리스트

### 1. 데이터 관리 작업
1. 데이터 매니저
  - 역할
    - 일부 CSV 파일 읽어오기 (일부는 데이터 매니저 없이 전용 스크립트에서 SO에 덮어씌움)
    - 게임 플레이 중 필요한 정보를 저장하고 불러오며, 리셋함
  - CSV 파일 읽기
    - CSV 파일 읽는 절차
      - DataManager.cs : 데이터 중앙 관리자 스크립트, 싱글톤
      - CSVReader.cs : 쉼표로 구성된 원본 CSV 파일을 게임 시스템이 이해할 수 있도록 쪼개고 읽는 스크립트
      - 각 Data.cs : 읽을 수 있도록 쪼개진 CSV 파일의 Row(가로 한 줄)의 데이터를 필요한 속성으로 담는 스크립트 
      - 각 Table.cs : 각 Data.cs를 여러 개 모아 관리하면서, 필요할 때 ID를 통해 해당 기능을 찾아주는 스크립트
      - 각 Controller.cs : ID를 입력받아, 각 Data.cs로 변환된 데이터를 바탕으로 대상의 제어 스크립트의 속성을 덮어쓰는 스크립트, 실제 대상의 컴포넌트로 붙음(MonoBehavior 상속)
    - 대상 : 플레이어 1종 스텟(HP, 공격력, 특수 공격력, 회복력 등), 몬스터 4종 스텟(ID랑 HP만 있음), 스테이지 이벤트, 스테이지 이벤트 보상
  - 게임 플레이 중 정보 저장, 불러오기, 리셋
    - 필요성 : (작성 중)

2. SO
  - 필요성 : 데이터 매니저는 플레이 모드 중에만 실행되는데, 에디터 단계에서 데이터를 변경해야하는 시스템이 있어서 필요
  - 관리 방법
