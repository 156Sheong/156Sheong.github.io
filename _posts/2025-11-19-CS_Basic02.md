## B. 컴퓨터가 이해하는 언어
1. 컴퓨터가 이해할 수 있는 언어 : 이진수(0, 1)로 된 기계어
   - 0, 1 인 이유는 스위치를 신호 없음(0) / 신호 있음(1) 으로 구분하기 때문
   - 약한 신호(0.25), 중간 신호(0.5), 강한 신호(0.75) 등이 사이에 없는 이유
     - 2가지 상태인 것이 물리적으로 간단하여 회로를 구현하기 쉽기 때문
     - 신호 손실, 신호 간섭, 잡음 등으로 인한 오류를 최소화할 수 있어, 신뢰성을 확보할 수 있기 때문
     - 논리 회로가 수학적 논리(불 대수)와의 완벽히 대응하여 수학으로 물리적인 하드웨어의 동작을 완벽히 설명하고 예측하며 설계할 수 있기 때문
     - 이미 이진법으로 체계가 잡혔는데, 다른 진법으로 바꾸면 인프라를 모두 다시 만들어야 함(경로의존성)
1. 그러나 사람은 이진수를 이해할 수 없다는 문제가 발생함
2. 따라서 사람은 고급 언어로 소스 코드를 작성하고, 컴퓨터는 이를 변환 과정을 거쳐 기계어로 기능을 수행함

혹시 높은 언어는 낮은 언어로 번역된다는 이야기를 들었는데,
 소스코드 -> 컴파일 -> 오브젝트 코드의 과정에서 '어셈블리어'는 컴파일 과정에서 포함되는 개념일까요?
아니면 별개의 개념일까요?


각 문자 인코딩 표준 별로 알파벳 문자를 표시하는 방법


비주얼 스튜디오 코드는 통합개발 환경인가?
 -> 아닙니다. 비주얼 스튜디오 코드는 '소스 코드 편집기'이며, 비주얼 스튜디오는 '통합 개발 환경(IDE)'입니다
 -> 그럼 통합 개발 환경과 소스 코드 편집기의 차이는 무엇인가?
 -> 소스 코드 편집기는 텍스트 기반의 코드 작성이 주 기능인 반면, 통합 개발 환경(IDE)은 코드 편집기 외에도 컴파일, 디버깅, 빌드, 버전 관리 등 개발에 필요한 모든 도구를 통합하여 제공합니다. 즉, 소스 코드 편집기가 IDE의 구성 요소 중 하나이며, IDE는 더 넓은 범위의 개발 과정을 지원하는 종합적인 환경
