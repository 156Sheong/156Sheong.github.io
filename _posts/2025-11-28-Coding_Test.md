# 코딩 테스트 입문
- 학습 목표
  - 실력 향상과 취업 준비를 위해 코딩 테스트의 개념과 목적, 준비 방법 등을 이해하는 것
  - 코딩 테스트 플랫폼에서 문제를 찾아 풀 수 있으며, 문제 해결 과정의 중요성을 이해하는 것

- 작성 툴 : Typora, Visual Studio Code

## B. 코딩 테스트란
- 지원자가 특정 프로그래밍 언어를 사용하여 주어진 문제를 제한 시간 내에 해결하는 시험
- 지원자의 문제 해결 능력, 논리적 사고력, 알고리즘 및 자로 구조 이해도를 평가하기 위한 시험
- 복잡한 문제를 작은 단위로 나누고(문제 분해), 효율적인 해결책을 설계하며(알고리즘), 이를 코드로 구현하는 능력을 종합적으로 평가함



## C. 코딩 테스트를 하는 이유
- **기업의 측면**
  - **지원자의 객관적 역량의 지표**
    - 서류 전형만으로는 지원자의 실제 프로그래밍 역량을 판단하기 어려운데, 코딩 테스트가 정량적인 지표를 제공해줌
  - **지원자의 문제 해결 능력과 논리력 확인**
    - 현업 개발 업무에선 끊임없이 새로운 문제가 발생하고 이를 해결해야 함, 코딩 테스트는 지원자가 낯선 문제를 접했을 때 어떻게 접근하고 해결하는지 과정을 알려줌
      - 지원자의 자료 구조와 알고리즘에 대한 이해도를 파악 가능
      - 같은 기능을 구현하더라도, 지원자가 얼마나 효율적인 코드를 작성하는지도 확인 가능

- **지원자의 측면**

  - **꾸준한 하고 체계적인 학습**
    - 코딩 테스트를 위해 자신에게 알맞은 단계에서 지속적으로 문제를 풀면서 계속 학습할 수 있음
  - **다양한 문제 경험**
    - 혼자서는 생각하기 어려운 다양한 유형의 문제를 접하면서, 새로운 문제를 만났을 때도 해결하는 방법을 찾을 수 있음
  - **자기개발 동기 부여**
    - 테스트 준비 과정에서 자신이 부족한 점을 발견하고 채워나가며 프로그래머로서 성장할 동기를 부여함 
  - **즉각적인 피드백**
    - 내가 작성한 코드가 정답인지 바로 확인 가능, 틀렸을 때도 바로 피드백을 받아 수정하는 과정에서 지원자가 성장함
  - **객관적인 실력 증력 및 어필**
    - 코딩 테스트는 코딩 실력 하나로 승부하기에, 지원자 입장에서 서류상 정보만으로는 보여주기 어려운 실질적 문제 해결 능력을 객관적으로 증명할 수 있는 기회가 됨, 그리고 높은 코딩 테스트 성적은 면접 과정에서 지원자에게 큰 자심감 부여
  - **기업 및 직무 적합성 판단**
    - 코딩 테스트 준비 과정과 실제 시험을 통해 지원자는 기업이 요구하는 기술 수준과 문화가 자신과 맞는지 간접 판단 가능
  - **트렌드 확인**
    - 기업은 최신 알고리즘 트렌드나 특정 라이브러리 활용 능력을 요구하는 문제를 자주 출제하기에, 지원자는 테스트 준비를 통해 최신 기술 동향 파악 가능
  - **성실성 증명**
    - 코등 테스트를 준비하면서 이를 공유하면, 기업에서 지원자의 성실성을 확인할 수 있는 지표로 활용 가능



## D. 코딩 테스트를 준비할 수 있는 플랫폼
- [프로그래머스](https://school.programmers.co.kr/learn/challenges?order=recent)
  - 실제 기업 코딩 테스트 환경과 유사한 UI / UX 를 제공하는 플랫폼
  - 기업별 기출문제나 모의 테스트도 제공
  - 초보자 친화적 UX / UI를 가짐
  - 문제가 난이도별로 잘 분류됨
    - 프로그래머스 난이도 기준 1~2 단계는 잘 풀어야함
    - 일부 대기업은 3단계를 요구하기도 함
  - 상세한 문제 설명과 예시 제공
- [백준 온라인 저지](https://www.acmicpc.net/)
  - 방대한 문제를 제공하여 국내 사용자들에게 인기가 높은 플랫폼
  - 다양한 유형과 난이도의 좋은 문제를 많이 보유함
  - 백지 상태에서 using, class 선언 등 다 해야하기 때문에 *초보자일 때는 추천하지 않음*
- [SWEA(SW Expert Academy)](https://swexpertacademy.com/main/main.do)
  - 삼성전자에서 주관하는 SW 역량 테스트 플랫폼
  - 삼성 입사를 목표로 한다면 유용함
- [LeetCode](https://leetcode.com/)
  - 글로벌에서 가장 잘 알려진 코딩 테스트 연습 플랫폼 중 하나
  - 글로벌 빅테크 기업 취업을 모ㅗㄱ표로 한다면 필수적인 플랫폼
  - 해외 기업의 출제와 문제 풀이 경향을 파악 가능



## E. 코딩 테스트 준비 방법
- **기초 문법 학습** : 자신이 선택한 언어의 기본 문법은 확실히 익혀야 함
- **자료 구조 및 알고리즘 개념 이해** - 배열, 스택, 큐, 트리, 그래프 등 핵심 자료 구조와 정렬, 탐색, 동적 계획법 등 주요 알고리즘 개념을 학습해야 함
- **문제 풀이 연습(핵심)**
  - **일단 시작**하는 것이 가장 중요함
  - 매일 일정한 시간을 할애해 **꾸준히** 문제를 풀어야 함, 매일 꾸준히 푸는 것이 하루 몰아서 여러 개 푸는 것 보다 실력이 향상됨
  - 여러 플랫폼을 활용해 **다양한 유형과 난이도의 문제를 경험**하기
  - 문제를 해결한 후 **다른 사람의 더 효율적인 코드를 보며 사고의 폭을 넓히는 코드 리뷰** 필요
  - 같은 유형의 문제를 **여러 번 반복해 풀며** 패턴 익히기 필요
  - 실제 시험처럼 **시간 제한**을 두고 문제를 풀어봐야 함 (30~60분 정도)
  - 어려웠던 부분이나 새로 배운 내용은 **메모하기**
- **전략적 접근**
  - 문제의 핵심 키워드를 파악하여 어떤 알고리즘이 적합할지 빠르게 유추하는 연습 필요
  - 문제를 풀기 전에 순서도 제작, 해결 아이디어 정리 등을 먼저 하기
  - 자신이 자주 틀리는 유형이나 약점을 보완하는 데 집중
- **약간의 팁**
  - 초반 연습 단계에서 오류가 발생했을 때, 비주얼 스튜디오를 이용하면 좀 더 편하게 오류를 찾을 수 있음
  - 코드의 화려함 보다는 가독성과 최적화가 중요함
  - 유니티 프로그래밍을 배우는 단계라면 C#을 우선시 해야하지만, `C#`이 익숙해졌을 때 `C++`도 도전한다면 더 나은 기회를 잡을 수 있음
  - 크롬 확장 프로그램인 [백준허브](https://chromewebstore.google.com/detail/%EB%B0%B1%EC%A4%80%ED%97%88%EB%B8%8Cbaekjoonhub/ccammcjdkpgjmcpijpahlehmapgmphmk)를 활용한다면, `프로그래머스`, `백준 온라인 저지`에서 푼 문제를 `깃허브`에 연동 가능함
