## B. 함수에 대하여
- 함수는 특정 작업을 수행하도록 미리 만들어둔 코드의 묶음

함수를 사용하는 이유
- 중복 코드 제거 (재사용성)
- 코드 가독성 증가 (=특정 동작을 명확하게 구분 가능)
- 유지 보수가 편함

- 재사용성
- 모듈화
- 추상화
- 타입 안정성
- 스코프 및 캡슐화

함수의 기본 구조
- 매개변수, 인자값
- 반환값

함수의 형태
- 반환값 X, 매개변수 X
void 함수명()
{
    // 코드
}

- 반환값 X, 매개변수 O
void 함수명(자료형 매개변수)    // 자료형과 매개변수의 개수는 매우 많이 넣을 수 있으나, 가독성 이슈가 있어서 많이 넣지는 않음
{
    // 코드
}

- 반환값 O, 매개변수 X
자료형 함수명()
{
    // 코드
    return 매개변수;
}


- 반환값 O, 매개변수 O
자료형 함수명(자료형 매개변수)
{
    // 코드
    return 매개변수;
}



심화 : 함수와 메서드의 차이
static, dynamic

Call by Value (값 타입)
- 함수가 변수 값을 복사해 가져감
- 함수 안에서 그 값을 바꿔도 원본을 해치지 않음
- out, ref 키워드 사용하면 값 자체도 직접 수정 가능

Call by Reference (참조 타입)
- 함수가 변수 값의 원본을 넘기는 것
- 함수 안에서 그 값을 바꾸면, 원본도 바뀜
- 


디버깅과 프로파일러
디버깅 방법 
- 중단점 기준으로 디버깅 시작 (비주얼 스튜디오 단축키 : F5)
- 함수를 통째로 실행하는 방법 (비주얼 스튜디오 단축키 : F10)
- 함수의 한 줄 한 줄을 실행하는 방법 (비주얼 스튜디오 단축키 : F11)
