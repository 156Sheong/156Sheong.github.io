# 프로그래머의 자기소개서 준비 방법
- 학습 목표
  - 취업용 자기소개서 작성의 준비 방법을 파악할 수 있음
  - 개발자를 꿈꾸게 된 `계기`와 `동기`를 명확히 정리할 수 있음
  - 현재 `자신의 상황`과 `강점`, `보완할 점`을 객관적으로 파악할 수 있음
  - 교육 과정에서 달성하고 싶은 `구체적 목표`를 설정할 수 있음

- 최종 수정일 : 2025-12-06

- 작성 툴 : Typora, Visual Studio Code

## B. 자기소개서의 정의
- 이력서가 지원자가 무엇을 했는지 개괄적으로 보여준다면, 자기 소개서는 어떻게 성장했고, 어떤 경험을 통해 무엇을 배웠으며, 그 역량을 어떻게 발휘할 수 있는지 등을 구체적으로 풀어내는 글
- 지원자의 성장 배경, 성격 장담점, 지원 동기, 직무 역량, 입사 후 포부 등을 구조화하여 자신의 삶의 가치관과 회사에 대한 이해도를 보여주는 글



## C. 자기소개서를 작성하는 이유
### Ca. 지원자의 측면
- **나의 매력 어필 및 차별화**
  - 이력서의 단편적 사실 정보만으로는 알 수 없는 지원자만의 경험, 가치관, 잠재력을 깊이 있게 보여줄 수 있음
- **면접의 나침반 역할**
  - 자소서에 쓴 내용이 면접의 질문으로 이어지므로, 면접관이 궁금해할 만한 포인트를 심어야 함
- **긍정적 동기 부여 및 자기 성찰**
  - 자기소개서를 작성하며 자기 자신과 회사에 대한 이해를 키워가며 취업에 한 걸음 다가갈 수 있음 
  - 취업 준비 과정에서 자신의 성과를 되돌아보며 부족한 점 파악 가능

### Cb. 기업의 측면
- **지원자에 대한 깊이 있는 이해**
  - 이력서의 단편적 정보만으로 알 수 없는 지원자의 가치관, 성격, 조직 적응력, 문제 해결 능력 등 파악 가능
- **직무 적합성 판단**
  - 지원 직무에 대한 지원자의 이해도와 해당 업무 수행 역량이 있는지 확인 가능
- **기업의 인재상 부합 여부 확인**
  - 지원자의 성장 배경, 가치관이 회사 문화와 잘 맞는지 적합성 판당 가능
- **서류 신뢰도 평가**
  - 이력서와 자소서 내용의 연결고리를 통해 지원자의 진정성과 서류 신뢰도 평가 가능



## D. 자기소개서에 들아가야 할 내용
- **자신에 대한 소개**
  - 자신이 어떤 일을 해 왔고, 살아온 경험을 통해 얻은 자신의 가치관과 성과를 표현해야 함
- **개발자가 되고 싶은 이유**
  - 왜 그 회사의 게임 프로그래머가 되고 싶은지 솔직하고 진정성 있게 작성해야 함
  - 들어갈 수 있는 항목 예시
    - 게임 프로그래밍에 관심을 갖게 된 계기
    - 어떤 게임을 플레이하면서 "이걸 만든 사람은 어떻게, 그리고 왜 이렇게 만들었을까" 궁금해서 탐구했던 경험
    - 게임 프로그래밍 자체에 매력을 느꼈던 순간
    - 개발자라는 직업의 어떤 점이 매력적으로 느껴졌는가
- **나에 대한 객관적 평가**
  - 나 스스로를 객관적으로 평가하고, 부족함 정을 인정할 수 있어야 함 (추후 이는 보완할 수 있어여)
  - 들어갈 수 있는 항목 예시
    - 프로그래밍 경험
      - 프로그래밍을 해본 경험이 있다면 어떤 언어로 무엇을 만들었는지
      - 없다면 완전 처음 시작하는 것에 대해 어떻게 느끼는지, 그리고 어떤것을 해서 어떤 역량을 키울 예정인지
    - 학습 스타일
      - 새로운 것을 배울 때 어떤 방식으로 배우는지
      - 혼자 공부하는 것을 선호하는지, 함께 하는 것을 선호하는지
      - 어려운 문제를 만났을 때 어떻게 해결하는지
    - 강점과 보완점
      - 자신의 강점은 무엇이고 그 사례는 무엇인지
      - 자신이 개선하고 싶은 부분은 무엇이고 어떻게 개선했는지
- **교육과정을 통해 이루고 싶은 것**
  - 교육과정이 끝났을 때 어떤 모습일지 스스로 현실적으로 상상해봐야 함
    - 기술적 목표
      - C#과 유니티를 얼마나 능숙하게 다룰 수 있게 되고 싶은지
      - 어떤 규모에서 어떤 장르의 게임을 만들고 싶은지
      - 특정 기술이나 시스템 구현에 관심이 있는지(AI, 네트워크, 그래픽스 등)
    - 학습 습관 목표
      - 어떤 학습 습관을 만들고 싶은지
      - 기술 블로그는 얼마나 자주 작성하고 싶은지
      - 코딩 테스트 문제는 얼마나 자주, 어느 정도 수준까지 풀고 싶은지
    - 진로 목표
      - 과정이 끝난 후 어떤 개발자가 되고 싶은지(장르, 규모, 파트 등)



## E. 핵심 작성 전략
- **지원 동기 및 직무 연관성**
  - 회사와 직무에 대한 철저한 조사 후, 왜 이 회사의 이 직무여야만 하는지 명확한 이유 제시
- **구체적이고 간결하게**
  - 추상적 미사여구 대신 두괄식으로 핵심 먼저
  - 경험은 STAR 기법(상황, 과제, 행동, 결과) 등으로 작성
- **솔직함과 진정성**
  - 자신의 경험과 역량을 바탕으로 과장없이 솔직하고 진정성 있게 작성

## G. 작성 팁
- **초고가 완벽할 필요 없음**
  - 초고는 원래 완벽하지 않은 것
  - 아직은 문장이 매끄럽지 않고, 문법이 틀려도 됨
  - 솔직한 것이 완벽한 것 보다 중요
  - 이력서 제출 전까지는 나중에 언제든 수정 가능
- **구체적으로 작성**
  - 예시 : 수업 후 매일 n 시간 씩 공부하고, 코딩 테스트는 하루에 얼마 정도 풀며, TIL은 가급적 매일 작성하겠다 등
- **긍정적이되 현실적으로 작성**
  - 달성 불가능한 목표 보다는 도전적이지만 현실적 목표 측정 필요
  - 예시 : 과정 종료 시 포트폴리오용 게임 2개 완성 등
- **정기적 업데이트 필요**
  - 한달에 한 번씩 다시 읽고 달라진 점 수정 필요
  - 새로운 목표가 생기면 추가하고, 달성한 목표는 체크하기
  - 과정 중간중간 소감 추가하는 것도 좋음



## H. 보관 및 활용 방법
- **저장 위치**
  - 기술 블로그에 포스팅으로 작성(비공개 / 공개 무관)
  - 깃허브에 private 레파토리로 관리
  - 로컬 컴퓨터의 바탕화면이나 접근 쉬운 폴더에 저장(바탕화면 추천)
- **활용 방법**
  - 월간 회고 시 : 한 달에 한 번씩 읽어보며 목표달성도 확인
  - 힘들 때 : 초심을 되찾고 동기부여를 얻기 위해
  - 중간 점검 시 : 3개월, 6개월 시점에 성장한 자신의 모습 확인
  - 과정 종료 시 : 처음과 끝을 비교하며 성장 스토리 정리
  - 취업 준비 시 : 실제 자기소개서나 면접 답변의 기반으로 활용
