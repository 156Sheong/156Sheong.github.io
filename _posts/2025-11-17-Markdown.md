## B. 마크다운 요약
- 마크다운 : 일반 텍스트를 사용하여 서식이 있는 문서를 쉽게 작성할 수 있도록 만들어진 `경량 마크업 언어`의 일종
  - `마크업 언어` : 문서의 구조와 서식을 정의하기 위해 태그에 둘러쌓인 언어, <Body>, <h1>, <section> 형식으로 작성됨
  
  - `경량 마크업 언어` : 기존의 마크업 언어 대신, 일반적인 텍스트 편집기를 사용하여 쉽게 작성하고 읽기 쉽도록 몇가지 기호들로 작성하도록 설계된 언어

  - 경량 마크업 언어의 종류 : AsciiDoc, BBCode, Textile, Markdown(마크다운) 등 
  
    
  
- 마크다운의 주요 특징
  - 쉬운 문법 : 일반 텍스트를 기반으로 하여 문법이 간단하고 직관적
  
  - 가독성 : 일반 텍스트처럼 읽고 쓰기 쉬움

  - 호환성 : GitHub의 문서, README 파일, 위키, 블로그 포스팅 등 다양한 웹 기반 콘텐츠에서 호환됨
  
    
  
- 마크다운을 사용하는 이유 : 프로그래머는 기술 문서를 자주 작성해야하는데, 마크다운을 쓰면 문서 작성 속도가 매우 빨라짐.



## C. 헤더 문법
- 헤더 : \# (H1) ~ \#\#\#\#\#\# (H6) 까지 6단계 헤더 가능
- 헤더 예시
# H1 헤더
## H2 헤더
### H3 헤더
#### H4 헤더
##### H5 헤더
###### H6 헤더



## D. 글씨 형태 문법

- **굵은 글씨** : 내용을 \*\*로 감싸면 굵은 글씨가 됨

  - 형태 : \*\*{쓸 내용}\*\*

    

- *이탤릭* : 내용을 \*로 감싸면 기울어진 이탤릭 글씨가 됨

  - 형태 : \*{쓸 내용}\*

    

- ***굵은 이탤릭*** : 내용을 \*\*\*로 감싸면 굵은 이탤릭 글씨가 됨

  - 형태 : \*\*\*{쓸 내용}\*\*\*

    

- ~~중간선~~ : 내용을 \~\~로 감싸면 중간선이 나타남

  - 형태 : \~\~{쓸 내용}\~\~




## E. 강조 문법

- `인라인 코드` : 내용을 \`로 감싸면 인라인 코드가 됨

  - 형태 : \`{쓸 내용}\`

  - 예시 : 플레이어가 피해를 입을 때는 `player.Hp` 변수의 값을 변화시키면 됩니다.

    

- <mark>Mark 태그</mark> : 내용을 \<mark>, \</mark>로 감싸면 Mark 태그가 됨
  
  - 형태 : \<mark>{쓸 내용}\</mark>
  
  - <mark>강조 내용</mark> 과 일반 내용



## F. 탈출 문자
- 탈출 문자(이스케이프 문자) : 프로그래밍에서 특정 문자가 원래 의미를 무시하고, 그 뒤에 오는 문자를 특수하게 해석하도록 하는 문자
- 마크다운의 탈출 문자 기호 : \
- 마크다운에서 탈출 문자의 의미 : 특수한 기능이 있는 문자를 일반 텍스트로 표시하기 위해 사용



## G. 코드 블럭 사용법

- 코드 블럭 사용법 : 내용의 윗줄과 아랫줄을 \```로 감싸면 코드 블럭이 됨

  - 형태

    - \```

    - {코드 내용}

    - \```

      

  - 예시

```
// 이 안에 코드가 적힙니다.
player.Hp -= 10;
using System;

class Program
{
    static void main(string[] args)
    {
        console.writeLine("Hello World");
    }
}
```



## H. 링크 사용법

- URL 직접 입력 : 주소를 직접 입력하는 방법
  - https://www.google.co.jp
  - www.google.co.jp
  
- 텍스트에 링크 삽입 : 텍스트에 링크를 삽입하는 방법
  - 형태 : \[구글](https\://www\.google\.co.jp)
  - 예시 : [구글](https://www.google.co.jp)

- 링크에 타이틀 설명 추가 : 마우스 포인터를 올리면 추가 설명이 있는 텍스트에 링크 삽입 방법
  - 형태 : \[구글](https\://www\.google\.co.jp "가장 보편적인 검색 엔진")
  - 예시 : [구글](https://www.google.co.jp "구글 링크 이동")

- 참조 링크 : 같은 링크 URL을 여러 번 입력하거나, 글 안의 링크를 따로 관리할 때 사용하는 방법
  - 형태
    - [구글][구글 링크1]
    - 검색 엔진 [구글 링크1]은 가장 보편적인 검색 엔진이다.
    - 검색 엔진 [구글 링크2]은 가장 보편적인 검색 엔진이다.
    - \[구글 링크1]: https\://www\.google\.co.jp
    - \[구글 링크2]: https\://www\.google\.co.jp "구글 링크 이동"
  
  - 예시
    - [구글][구글 링크1]
    - 검색 엔진 [구글 링크1]은 가장 보편적인 검색 엔진이다.
    - 검색 엔진 [구글 링크2]은 가장 보편적인 검색 엔진이다.
  [구글 링크1]: https://www.google.co.jp
  [구글 링크2]: https://www.google.co.jp "구글 링크 이동"

## J. 마크다운 사용 유의사항
- \<, \> 와 같은 기호 사용이 어려움
  - 대안1 : 큰따옴표(" ") 또는 작은 따옴표(' ') 사용
    - 사용 예시 : 도서 "컴퓨터 밑바닥의 비밀"
    - 이슈 : 기호가 작아서 눈에 잘 안 띄는 경우가 있음

  - 대안2 : 겹낫표 (『 』) 또는 낫표 (「 」) 사용
    - 사용 예시 : 도서 『컴퓨터 밑바닥의 비밀』
    - 이슈 : Windows 키 + . (마침표) or Windows 키 + ; (세미콜론)을 동시에 눌러 이모지를 연 뒤, 찾아야 해서 번거로움
  
  - 대안3 : \ 기호(탈출문자) 사용
    - 사용 예시 : 도서 \<컴퓨터 밑바닥의 비밀\>
    - 이슈 :  \, \< 2개의 기호를 동시에 사용해야 하는 것이 번거로울 수 있음

- 플랫폼별 호환성 주의 : 마크다운의 랜더링 결과는 사용하는 플랫폼에 따라 달라질 수 있음
