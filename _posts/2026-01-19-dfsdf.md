FixedUpdate 물리 연산을 위해 사용
FixedUpdate 는 특정 프레임에서만 호출되기에, 키 입력은 그냥 Update에서 받아야 함(버튼 눌렀는데 FixedUpdate가 호출되지 않으면 입력이 먹히지 않음)

ForceMdoe
  Force:
  특징: 질량을 고려하여 지속적으로 힘을 가합니다 (자동차 엑셀처럼 가속).
  용도: 물체가 무게를 가지고 부드럽게 가속될 때 사용 (예: 물건 밀기).
  Acceleration:
  특징: 질량을 무시하고, 힘의 크기만큼 지속적으로 가속시킵니다.
  용도: 질량과 상관없이 특정 속도로 계속 밀어주고 싶을 때 (예: 추진기).
  Impulse:
  특징: 질량을 고려하여 순간적으로 힘을 가해 충격량(Momentum)을 변화시킵니다.
  용도: 순간적인 점프, 충돌 시 튕겨 나가는 효과 (예: 점프).
  VelocityChange:
  특징: 질량을 무시하고 순간적으로 속도(Velocity)를 변경합니다.
  용도: 질량과 상관없이 즉각적인 속도 변화가 필요할 때

충돌처리
Collision
Collider 컴포넌트의 'IsTrigger'가 선택해제 되어있어야 합니다.
자신과 충돌 대상 모두에 Rigidbody 컴포넌트가 추가되어 있어야 하고, 두 Rigidbody중 하나는 IsKinematic이 체크 해제되어 있어야 합니다.
활성화 된 오브젝트 간의 충돌처리에 사용합니다.

Tirgger
Collider 컴포넌트의 'IsTrigger'가 선택되어 있어야 합니다.
IsTrigger 선택시 해당 오브젝트는 물리적인 충돌이 진행되지 않고 통과됩니다.
자신이나 충돌 대상중 하나에 Rigidbody 컴포넌트가 추가되어 있어야 합니다.
주로 특정 장소 도달 등의 상황에 사용됩니다.
OnCollisionEnter(Collision other) 
OnTriggerEnter(Collider other)    // Enter : 충돌이 실행될 때 1회 실행됩니다.

OnCollisionStay(Collision other)  
OnTriggerStay(Collider other)     // Stay : 충돌이 유지되는 동안 반복적으로 실행됩니다, 상대적으로 적게 사용됨

OnCollisionExit(Coliision other)  
OnTriggerExit(Collider other)     // Exit : 충돌이 종료될 때(벗어나는 순간) 1회 실행됩니다.
