## B. 알고리즘 이란?
- 알고리즘이란 문제를 해결하기 위한 과정을 논리적 절차에 따라 구성한 일련의 단계
- 모든 입력값에 대해 튜링머신이 정지하게 하는 명령
- 입력값에 따른 결과가 도출됨
- 알고리즘은 다음과 같은 일련의 과정을 거침
  - **1. 문제 이해** -> **2. 예시와 테스트 케이스 작성** -> **3. 알고리즘 설계** -> **4. 알고리즘 구현 및 검증** -> **5. 알고리즘 분석과 개선의 과정**


## C. 알고리즘 풀이의 각 단계
1. **문제 이해**
   - 우선, 문제의 요구사항을 분석하고 입력과 출력의 형식을 파악해야 함
   - **입력 크기**, **제한 시간** 등도 파악해야 함
   - 문제의 핵심 수식과 논리만 남기도록 **추상화** 필요
2. **예시와 테스트 케이스 작성**
   - 어떤 입력 값을 넣었을 때 어떤 출력이 나올지 몇가지 예시를 파악해야 함
   - **경계 값(입력이 최소 값이나 최대 값)**과 **예외 처리**, **특이 케이스(중복 값, 음수, 비정렬 데이터 등)** 등을 고려해야 함
3. **알고리즘 설계**
   - 알고리즘 설계에서 가장 중요한 것은 **정확성**
   - 그리고 이를 보증할 **검증 방법**도 중요
     - 알고리즘의 효율성(BigO)과 개선은 구현 및 검증 이후에 해도 됨
   - 설계 단계에선 **플로우 차트**나 **수도 코드**를 이용할 수도 있음  
4. **알고리즘 구현 및 검증**
  - 실제 코드를 작성하는 단계
  - 사전에 만들어 둔 테스트 케이스로 검증 필요
  - 이 단계에서 적절한 주석을 넣으면 나중에 코드를 다시 파악하고 수정아기 편해짐
  - 정상적으로 작동하지 않으면, 다시 테스트 케이스를 작성하고 설계를 해야 함
5. **알고리즘 분석과 개선의 과정**
  - 단순 맞추는 것이 중요한 것이 아니라, **내 코드를 회고** 하는 과정이 필요
  - 코딩 테스트 처럼, 같은 문제를 다른 알고리즘으로 푼 사례가 있다면 참조하며 공부해야 함
  - **리펙토링**을 통해 코드를 더 직관적으로 바꾸거나, 중복된 코드를 제거하며, 기능별로 **모듈화** 하면서 개선해야 함


## D. 자주 사용되는 알고리즘 유형

## E. 알고리즘의 복잡도
- 알고리즘의 복잡도에는 **시간 복잡도**, **공간 복잡도** 가 있음
- 복잡도가 낮을수록 더 효율적인 알고리즘
- **시간 복잡도(Time Complexity)**
  - 알고리즘이 문제를 해결하는 데 걸리는 시간의 양
  - 연산 횟수를 기준으로 측정하고, 입력 데이터의 크기(n)이 커짐에 따라 실행 시간이 얼마나 빠르게 증가하는지를 파악
- **공간 복잡도(Space Complexity)**
  - 알고리즘을 실행하는 데 필요한 총 메모리 공간의 양
  - 입력 크기에 따른 필요한 메모리 양의 증가율을 분석함


## F. 알고리즘의 복잡도를 표시하는 방법
- **Big-O 표기법**
  - 알고리즘 최악의 실행 시간을 표기
  - 최소한 보장되는 성능을 표기하기 때문에 가장 일반적으로 사용됨
- **Big-Ω 표기**
  - 법 알고리즘 최상의 실행 시간을 표기
- **Big-θ 표기법**
  - 알고리즘 평균 실행 시간을 표기
