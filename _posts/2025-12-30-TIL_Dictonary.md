## B. 해시 테이블(Hash Table) 이란?
- 하나의 키(Key)와 값(Value)을 연결해 저장하고, 키를 통해 연관 값을 얻을 수 있는 자료구조
- 키-값 쌍(Key-Value Pair)으로 배열을 만들어 데이터를 관리함
  - 입력되는 키 값의 해싱 결과 값은 배열의 인덱스를 기준으로 출력됨
- 각 키는 해싱(Hashing)되어 값을 가리키는 고유한 식별자 역할을 함
- 데이터 위치를 직접 계산해 접근하기 때문에, 대부분의 경우 삽입, 삭제, 읽기, 검색 연산이 O(1) 으로 빠름
- C# 에서는 **Dictionary<T>** 라는 **제네릭 클래스**로 구현됨


## C. 해시(Hash) 란?
- 임의의 길이를 가진 데이터를 **해시 알고리즘**을 사용해 고정된 길이를 가진 데이터로 출력한 값
- 입력된 키 값은 해싱한 후 저장될 위치를 가리키는 인덱스가 될 수 있음


### Ca. 해시의 특징
- **단방향성(One-wayness)**
  - 데이터를 해시 값으로 출력할 수 있지만, 반대로 해시 값을 다시 원래의 키 데이터로 돌리는 것은 매우 어려움
  - 따라서 해시는 비가역적이고, 민감한 정보의 보안을 강화하는데 사용할 수 있음
- **해시 충돌(Hash Collision) 가능성**
  - 해시 함수는 무한한 경우의 수를 가진 입력값에 대해 유한한 출력값을 생성해 충돌이 필연적
  - 좋은 해시 충돌 함수를 사용하여 해시 테이블 충돌을 최소화해야함
- **고정된 데이터의 길이**
  - 어떤 데이터를 삽입하더라도, 해싱의 결과물은 항상 고정된 길이의 해시 값 데이터를 출력함
  - 데이터 저장 시 값의 길이가 동일하면 메모리 공간을 효율적으로 관리 가능


## D. 해시 테이블의 구성 요소 설명
- **키**
  - 데이터를 저장하고 찾기 위해 사용되는 고유 식별자
  - 해시 함수를 거쳐 데이터를 보관할 수 있는 식별 코드(헤시 코드)로 전환 됨
- **값**
  - 키를 기준으로 한 위치에 저장되는 데이터
  - 키를 통해 값에 엑세스 가능
- **해시 함수**
  - 입력된 키를 해시 테이블의 주소(해시 코드)로 변환하는 함수
  - 해시 함수의 성능을 결정하는 중요 요소
    - 데이터를 읽기, 검색, 삽입, 삭제 모두에 해싱이 일어남
  - 보관하고자 하는 테이블의 크기 내의 결과를 출력


## E. 해시 테이블의 동작 원리와 주의 사항 설명
- 해시 테이블의 키와 값이 입력되면, 해시 함수가 키를 가공하여 해시 테이블에 값과 함께 저장
- 해시 함수가 키를 가공할 때, 충돌할 가능성이 생김
- 키 충돌을 최대한 방지하기 위한 몇 가지 대처법이 있음


### Ea. 키 충돌 방지 기법 종류
- (해시 테이블의 핵심이라 중요함)
- **체이닝 기법**
  - 해시 충돌 시, 같은 인덱스의 여러 데이터를 연결 리스트(Linked List) 형태로 연결해 저장하는 방법
- **개방 주소법 : 선형 탐사**
- 해시 충돌 시, 원래 주소의 다음 칸부터 순차적으로 빈 공간을 찾아 데이터를 저장하는 방법
- **개방 주소법 : 제곱 탐사**
  - 해시 충돌 시, 초기 해시 주소의 제곱수 만큼 떨어진 빈 슬롯을 순차적으로 탐색해 데이터를 저장하는 방법
- **개방 주소법 : 이중 해싱**
  - 2개의 서로 다른 해시 함수를 사용하여, 해시 충돌 시 해당 주소를 다시 해싱하여, 두 번째 해시 함수로 결정된 간격으로 다음 탐색 위치를 찾아가는 방법


## F. 해시 테이블이 유용한 경우
- ID로 회원 정보 찾기 처럼, 순서보다는 데이터 관계가 중요할 때
- 데이터를 빠르게 검색하고 싶을 때(배열이나 리스트 보다 성능 좋기 때문)
- 설정 값, 맵핑(서로 다른 데이터나 요소를 연결하고 대응시키는 과정) 정보처럼 키-값 구조로 데이터를 다루는 것이 자연스러울 때
- (보안에 특화된 알고리즘 사용 시) 키 값을 알기 어렵게 암호화할 때


## G. C#의 딕셔너리(Dictionary)
- C# 에서 해시 테이블이 **제네릭 클래스**로 구현된 것
- 딕셔너리는 클래스 내부적으로 2개의 배열로 되어있음
  - 값은 Entry 구조체 타입의 entries 배열
  - 키는 int 타입의 buckets 배열
