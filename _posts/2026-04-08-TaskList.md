## B. 작업 리스트

### 1. 데이터 관리 작업
1. 데이터 매니저
  - 역할
    - 일부 CSV 파일 읽어오기 (일부는 데이터 매니저 없이 전용 스크립트에서 SO에 덮어씌움)
    - 게임 플레이 중 필요한 정보를 저장하고 불러오며, 리셋함
  - CSV 파일 읽기
    - CSV 파일 읽는 절차
      - DataManager.cs : 데이터 중앙 관리자 스크립트, 싱글톤
      - CSVReader.cs : 쉼표로 구성된 원본 CSV 파일을 게임 시스템이 이해할 수 있도록 쪼개고 읽는 스크립트
      - 각 Data.cs : 읽을 수 있도록 쪼개진 CSV 파일의 Row(가로 한 줄)의 데이터를 필요한 속성으로 담는 스크립트 
      - 각 Table.cs : 각 Data.cs를 여러 개 모아 관리하면서, 필요할 때 ID를 통해 해당 기능을 찾아주는 스크립트
      - 각 Controller.cs : ID를 입력받아, 각 Data.cs로 변환된 데이터를 바탕으로 대상의 제어 스크립트의 속성을 덮어쓰는 스크립트, 실제 대상의 컴포넌트로 붙음(MonoBehavior 상속)
    - 대상 : 플레이어 1종 스텟(HP, 공격력, 특수 공격력, 회복력 등), 몬스터 4종 스텟(ID랑 HP만 있음), 스테이지 이벤트, 스테이지 이벤트 보상
  - 게임 플레이 중 정보 저장, 불러오기, 리셋
    - 필요성 : (작성 중)
    - 대상 기능
      - 플레이어 현재 스텟(전투 종료 후 남은 체력, 스테이지 이벤트로 얻는 스텟)
      - 플레이어 스킬 해금 여부(스테이지 이벤트로 얻음)

2. SO

다음은 이벤트 및 보상과 데이터 관리 기능입니다.


기능은 스테이지 / 전투 2개로 나눠서 관리
  스테이지 : 상황에 맞는 이벤트를 랜덤으로 불러오고, 유저가 선택한 보상을 저장
  전투
    시작 시 저장된 플레이어 및 몬스터 스텟 및 스킬 데이터 불러오기,
    전투 중 플레이어는 조건에 맞다면 스킬 발동,
    전투 종료 시 플레이어의 현재 스텟을 저장
  이 사이에 싱글톤의 데이터 매니저가 데이터 전달, 게임 재시작 시 스탯 및 보상 초기화
  

데이터는 관리가 용이하도록 CSV 파서를 사용해 딕셔너리 형태로 데이터 매니저에 저장,
엑셀을 활용해 수치만 변경하면 바로 적용
이상입니다.

데이터 불러오는 과정 :  데이터 테이블 -> 파서 -> 한줄 씩 로우 데이터를 테이블 스크립트에 저장 -> 데이터 매니저

(SO는 파서 필요 X, 메모리 부분에서 효율적 / 그러나 대량 수정이나 유니티 외부에서 편집이 불편)

개선하고 싶은 부분
플레이어와 몬스터의 스텟, 스킬 데이터가 서로 호환되지 않게 구현 -> 추후 개발할 때는 하나의 스텟과 스킬 조건 판단할 수 있는 컨트롤러를 구현하고 싶음
데이터 저장과 불러오는 기능이 데이터 매니저에 함께 있음, 따로 독립된 저장 기능으로 개선하고 싶음
