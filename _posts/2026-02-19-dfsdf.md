PlayerPrefs
- 로컬 데이터로 저장됨
- 저장 경로는 운영 체제에 따라 다름

- 사용처
  - 설정창
  - 튜토리얼 완료 여부

- 부적절한 경우
  - 중요한 정보는 부적절


CSV
- 컴마로 분리된 값들

- 사용 이유
  - SerializeField 로 선언하면 기획자 접근 불가(협업 불가)
  -   하드 코딩
  -   데이터 변경 용이하지 않음

- 이점
  - 게임 로직과 데이터의 분리 -> 기획자와 협업 및 데이터 값 수정 용이
  - 버전 관리 및 협업 용이

- 단점
  - 계층적 구조 표시 분리
  - ㅇㄹ

- 한계점
  - 타입 정보 없음(int, float, string 등)
  - 구조적 표현 불가
  - 런타임 수정 불편



Json
개요
  - 객ㄹ체를 텍스트로 바꾸어 외부와 교환하는 방법

실무에서 가지는 문제점
  - 하드코딩된 필드 변수
  - 메모리 휘발



유니티에서 제공하는 Json 2가지
- JsonUtility
- 특징
  - 유니티 기본 제공
  - 빠름
  - 제한 많음
- 장점
  - 간단한 구조 적합
  - 세이브 / 설정에서 충분히 활용 가능
- 한계점 -> 이 경우 Newtonsoft JSON 사용
  - 딕셔너리 미지원
  - 프로퍼티 미지원
  - 상속 제한

- Newtonsoft JSON
  - 상대적으로 무거운 경향이 있음
  - 기획자랑 협업 시 csv 가 더 나을 수 있음
  - 서버 통신 / 복잡한 데이터는 Newtonsoft JSON
  - 간단한 설정 / 캐싱 -> JsonUtility
