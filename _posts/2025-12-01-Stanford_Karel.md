## B. 스탠포드 카렐의 정의
- 컴퓨터 프로그래밍 입문자들에게 프로그래밍의 기본 개념을 가르치기 위해 설계된 교육용 프로그래밍 언어 겸 환경
- 프로그래밍 초보자가 복잡한 언어의에 얽매이지 않고, 순차, 반복, 조건 실행과 같은 핵심 아이디어를 배우도록 돕는 것이 목적
- 1970년대 스탠포드 대학의 대학원생 이었던 리처드 E. 패티스가 개발
- 카렐이라는 명칭은 `로봇`이라는 명칭을 처음 사용한 체코의 극작가 카렐 차페크(Karel Čapek)의 이름에서 따옴



## C. 스탠포드 카렐의 주요 특징
- 스탠포트 카렐은 **단순한 환경**과 **제한된 명령어**가 특징
- **그리드 기반 환경**
  - 카렐 로봇은 벽으로 둘러싸인 그리드 기반의 단순한 세계에서 움직임
- **제한된 기본 명령**
  - 로봇이 이해할 수 있는 기본 명령어가 매우 적음
    - move(): 한 칸 앞으로 이동
    - turnLeft(): 왼쪽으로 90도 회전
    - putBeeper(): 현재 위치에 '비퍼'(beeper, 데이터를 나타내는 물체)를 놓음
    - pickBeeper(): 현재 위치에 있는 비퍼를 줌
    - turnOff(): 프로그램 실행을 종료하고 로봇을 종료
- **추상화된 환경**
  - 로봇의 움직임과 환경 상호작용에만 초첨을 맞춤
  - 실제 프로그래밍 언어에서 접하는 데이터 구조, 변수, 숫자, 문자열 같은 복잡한 요소가 없음
- **조건문 및 반복문 지원**
  - 단순한 환경에도 if, while 등 조건 제어 흐름문과 반복 구조를 사용하여 문제 해결 가능
- **함수 정의 가능**
  - 사용자가 turnRight()와 같은 새로운 함수를 직접 정의하여 추상화 수준을 높이고 코드를 구조화 가능
- **다양한 언어 구현**
  - C++, 자바, 파이썬 등 다양한 프로그래밍 언어로 구현 가능 



## D. 스탠포드 카렐로 배울 수 있는 것과 문제 해결 접근법
- 카렐로 문제를 푸는 것은 실제 프로그래밍과 동일한 과정
- 명령을 순서대로 실행하는 순차 구조, 조건에 따른 조건문, 동작을 반복하는 반복문 모두 경험 가능
- 카렐이 움직이는 모습을 직접 눈으로 확인 가능해 코드 동작을 이해하기 쉬움
- 문제 해결 단계<br>
  1) **문제 파악하기**<br>
    - 카렐의 시작 위치, 방향, 목표 확인
  2) **계획 세우기**<br>
    - 필요한 동작을 순서대로 나열
  3) **코드 작성하기**<br>
    - 명령어를 순서대로 작성
  4) **실행하고 확인하기**<br>
    - 카렐이 의도대로 움직이는지 확인
